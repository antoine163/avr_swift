static void twi_receive_byte_nack ( )
inlinestatic

Prépare le composent à resevoire un octet inci qu'a trensmétre le bit nack.

Note
En mode esclave, l'appelle a cette fonction permet aussi de préparer la réception du statu TW_SR_STOP.
Si le mode maître est actifs.
Ceci signifie qu'un octet va être ressue venant d'un esclave, ensuit le mètre va écrira le bit nack, pour indiquer qu'il ne veut plu continuer à lire de données.

Le statut qui peut survenir après l'appelle de cette fonction sont :
  • TW_MR_DATA_NACK
    Les actions qui peuve être générai ensuit sont :
    twi_get_received_byte() suivi de twi_generate_start()
    twi_get_received_byte() suivi de twi_generate_stop()
    twi_get_received_byte() suivi de twi_generate_stop_start()
Si le mode esclave est actifs.
Ceci signifie qu'un octet va être ressue venant du maître, ensuit, l'esclave écrira le bit nack.

Les status qui peuve survenir après l'appelle de cette fonction sont :
  • TW_SR_DATA_NACK
  • TW_SR_GCALL_DATA_NACK
    Les actions qui peuve être générai ensuit sont :
    twi_get_received_byte() suivi de twi_receive_byte_ack() (commute dans le mode non adresser)
    twi_get_received_byte() suivi de twi_receive_byte_nack() (commute dans le mode non adresser)

  • TW_SR_STOP (commute dans le mode non adresser)
    Les actions qui peuve être générai ensuit sont :
    twi_receive_byte_ack()
    twi_receive_byte_nack()
